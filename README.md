# Гірські котеджі — статичний сайт

Готова структура для розміщення на будь-якому статичному хостингу (GitHub Pages, Netlify, FTP-хостинг).

## Структура
```
/images
    hero.jpg       — зображення на головному екрані
    /cottage1      — фото котеджу #1
    /cottage2      — фото котеджу #2
    /general       — загальні фото території
data/calendar-availability.json — опубліковані інтервали зайнятості котеджів
style.css      — стилі оформлення
script.js      — логіка меню, плавного скролу та галереї
favicon.svg    — значок сайту
index.html     — головна сторінка
robots.txt     — правила для пошукових роботів
```

## Що замінити
- **Контакти** у розділі «Контакти» (телефон, Telegram, Viber, email, Instagram).
- **Ціни та назви** котеджів у розділі «Наші котеджі».
- **Зображення**: додайте ваші фото у папки `images/cottage1`, `images/cottage2` та `images/general`.
- **Карту**: у Google Картах відкрийте вашу локацію → Поділитися → Вбудовування карти → замініть `src` у `<iframe>`.
- **Sitemap**: відредагуйте `robots.txt`, якщо додасте sitemap.

## Розгортання
- **GitHub Pages**: завантажте все в репозиторій, вмикайте Pages для гілки `release`, корінь `/` (або налаштуйте свою публічну гілку).
- **Netlify**: перетягніть папку в Netlify, або підключіть Git-репозиторій (build: нема, publish: корінь).
- **Звичайний хостинг**: завантажте в корінь домену через FTP.

## Календарі бронювання

Секретні iCal-посилання з Booking.com не потрапляють у репозиторій. Окрема GitHub Actions (`sync-calendars.yml`) кожні 30 хвилин завантажує iCal, прибирає всі службові поля й публікує лише інтервали зайнятості у файлі `data/calendar-availability.json`. Якщо інформація не змінилася — файл не перезаписується.

У репозиторії повинні бути налаштовані секрети:

- `BOOKING_CALENDAR_1` — посилання на календар котеджу №1.
- `BOOKING_CALENDAR_2` — посилання на календар котеджу №2.

Фронтенд читає `data/calendar-availability.json` і підсвічує зайняті дати у календарі без жодних згадок про реальні iCal-посилання.

### Як протестувати локально

1. Створіть тимчасові файли з тестовими подіями у форматі iCal (можна скачати реальні `.ics` з Booking.com або підготувати власні).
2. Запустіть скрипт оновлення, вказавши шляхи до цих файлів у змінних середовища:
   ```bash
   BOOKING_CALENDAR_1="file:///повний/шлях/до/cottage1.ics" \
   BOOKING_CALENDAR_2="file:///повний/шлях/до/cottage2.ics" \
   python3 scripts/update_calendar_availability.py
   ```
3. Переконайтесь, що згенерований файл `data/calendar-availability.json` містить лише очікувані інтервали зайнятості без чутливих полів.
4. Відкрийте `index.html` у браузері (або підніміть локальний HTTP-сервер) та перевірте підсвічування зайнятих дат у календарі.
