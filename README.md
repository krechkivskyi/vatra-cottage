# Гірські котеджі — статичний сайт

Готова структура для розміщення на будь-якому статичному хостингу (GitHub Pages, Netlify, FTP-хостинг).

## Структура
```
/images
    hero.jpg       — зображення на головному екрані
    /cottage1      — фото котеджу #1
    /cottage2      — фото котеджу #2
    /general       — загальні фото території
style.css      — стилі оформлення
script.js      — логіка меню, плавного скролу та галереї
favicon.svg    — значок сайту
index.html     — головна сторінка
robots.txt     — правила для пошукових роботів
calendar-config.sample.json — зразок конфігурації посилань календарів
```

## Що замінити
- **Контакти** у розділі «Контакти» (телефон, Telegram, Viber, email, Instagram).
- **Ціни та назви** котеджів у розділі «Наші котеджі».
- **Зображення**: додайте ваші фото у папки `images/cottage1`, `images/cottage2` та `images/general`.
- **Карту**: у Google Картах відкрийте вашу локацію → Поділитися → Вбудовування карти → замініть `src` у `<iframe>`.
- **Sitemap**: відредагуйте `robots.txt`, якщо додасте sitemap.

## Розгортання
- **GitHub Pages**: завантажте все в репозиторій, вмикайте Pages для гілки `release`, корінь `/` (або налаштуйте свою публічну гілку).
- **Netlify**: перетягніть папку в Netlify, або підключіть Git-репозиторій (build: нема, publish: корінь).
- **Звичайний хостинг**: завантажте в корінь домену через FTP.

## Календарі бронювання

Посилання на iCal-фіди підтягуються з файлу `calendar-config.json`, який створюється під час деплою зі секретів GitHub Actions. Для локального тестування можна створити файл `calendar-config.json`, скопіювавши `calendar-config.sample.json` і замінивши значення на власні.

У GitHub додайте секрети в репозиторії:

- `BOOKING_CALENDAR_1` — посилання на календар котеджу №1.
- `BOOKING_CALENDAR_2` — посилання на календар котеджу №2.

Під час виконання GitHub Actions буде згенеровано файл `calendar-config.json`, що не зберігається в репозиторії.
