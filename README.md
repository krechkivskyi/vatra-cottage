# Гірські котеджі — статичний сайт

Готова структура для розміщення на будь-якому статичному хостингу (GitHub Pages, Netlify, FTP-хостинг).

## Структура
```
/images
    hero.jpg       — зображення на головному екрані
    /cottage1      — фото котеджу #1
    /cottage2      — фото котеджу #2
    /general       — загальні фото території
style.css      — стилі оформлення
script.js      — логіка меню, плавного скролу та галереї
favicon.svg    — значок сайту
index.html     — головна сторінка
robots.txt     — правила для пошукових роботів
/data          — публічні JSON з інтервалами зайнятості (генеруються автоматично)
/scripts       — службові скрипти GitHub Actions
```

## Що замінити
- **Контакти** у розділі «Контакти» (телефон, Telegram, Viber, email, Instagram).
- **Ціни та назви** котеджів у розділі «Наші котеджі».
- **Зображення**: додайте ваші фото у папки `images/cottage1`, `images/cottage2` та `images/general`.
- **Карту**: у Google Картах відкрийте вашу локацію → Поділитися → Вбудовування карти → замініть `src` у `<iframe>`.
- **Sitemap**: відредагуйте `robots.txt`, якщо додасте sitemap.

## Розгортання
- **GitHub Pages**: завантажте все в репозиторій, вмикайте Pages для гілки `release`, корінь `/` (або налаштуйте свою публічну гілку).
- **Netlify**: перетягніть папку в Netlify, або підключіть Git-репозиторій (build: нема, publish: корінь).
- **Звичайний хостинг**: завантажте в корінь домену через FTP.

## Календарі бронювання

### Як працює інтеграція

- Приватні посилання iCal з Booking зберігаються у секретах репозиторію `BOOKING_CALENDAR_*`.
- За розкладом (кожні 30 хвилин) і вручну запускається GitHub Actions [`Update booking calendars`](.github/workflows/update-calendars.yml).
- Workflow виконує `npm run fetch-calendars`, який бере всі змінні середовища `BOOKING_CALENDAR_*`, завантажує відповідні iCal-фіди, видаляє чутливі поля й залишає тільки інтервали зайнятості.
- Для кожного календаря створюється публічний файл `data/busy-<name>.json` з такою структурою:

```json
{
  "generatedAt": "2024-05-20T15:05:00.000Z",
  "intervals": [
    { "start": "2024-05-21T12:00:00.000Z", "end": "2024-05-25T10:00:00.000Z", "allDay": false }
  ],
  "busyDates": ["2024-05-21", "2024-05-22", "2024-05-23", "2024-05-24"]
}
```

Фронтенд читає лише ці JSON-файли, тож жоден реальний iCal-URL не потрапляє у вихідний код, бандл чи мережеві запити браузера.

### Налаштування секретів

Додайте у репозиторій GitHub Secrets:

- `BOOKING_CALENDAR_1` — посилання на iCal котеджу №1;
- `BOOKING_CALENDAR_2` — посилання на iCal котеджу №2.

За потреби можна додавати більше календарів (`BOOKING_CALENDAR_3`, `BOOKING_CALENDAR_SUMMER` тощо) — скрипт згенерує `data/busy-3.json`, `data/busy-summer.json` і т.д. Не забудьте передати нові секрети в job (див. `env` у workflow).

### Локальне тестування

1. Встановіть залежності: `npm install`.
2. Підготуйте тестові iCal-файли або використайте тимчасові публічні посилання.
3. Запустіть скрипт, передавши шляхи/URL через змінні середовища:

   ```bash
   BOOKING_CALENDAR_1=./fixtures/cottage-1.ics \
   BOOKING_CALENDAR_2=https://example.com/test.ics \
   npm run fetch-calendars
   ```

   Локальні шляхи дозволені: скрипт зчитає файл з диску.

4. Переконайтесь, що у `data/` зʼявилися оновлені `busy-*.json`, після чого можна відкрити `index.html` у браузері або запустити локальний вебсервер (`npx http-server .`).

Під час помилок у логах виводиться лише коротке повідомлення зі статусом, без розкриття приватних URL.
